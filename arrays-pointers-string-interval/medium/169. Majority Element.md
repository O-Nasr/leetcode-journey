## Initial Thoughts
The **Boyer–Moore Majority Vote Algorithm** is a clever technique used to find the *majority element* in an array — that is, the element that appears more than ⌊n/2⌋ times. It’s especially valued because it runs in **linear time (O(n))** and uses **constant space (O(1))**, making it very efficient compared to other approaches like hashing or sorting.

### 🔑 How it works
1. **Candidate selection phase**  
   - Start with the first element as a *candidate* and set a counter to 1.  
   - As you iterate through the array:
     - If the next element matches the candidate, increment the counter.
     - If it doesn’t match, decrement the counter.
     - When the counter drops below 0, switch the candidate to the current element and reset the counter to 1.

2. **Verification phase (optional)**  
   - In problems where a majority element is guaranteed (like the LeetCode problem you’re viewing), the candidate at the end of the first pass is the majority element.  
   - If not guaranteed, you’d need a second pass to confirm that the candidate actually appears more than ⌊n/2⌋ times.

### ⚡ Example
Array: `[2,2,1,1,1,2,2]`  
- Candidate starts as `2`, count goes up and down as you iterate.  
- By the end, candidate = `2`, which is indeed the majority element.

### ✨ Why it’s powerful
- **Efficiency**: Only one pass through the array, no extra memory.  
- **Elegance**: Uses the idea that the majority element cannot be “canceled out” by minority elements.

```c#
public int MajorityElement(int[] nums) {
    int candidate = nums[0];
    int count = 1;

    for (int i = 1; i < nums.Length; i++)
    {
        if (candidate == nums[i])
        {
            count++;
        }
        else
        {
            count--;
            if (count < 0)
            {
                candidate = nums[i];
                count = 1;
            }
        }
    }
    
    return candidate;
}
```
