## Initial Thoughts

A number is "happy" if repeatedly replacing it with the sum of the squares of its digits eventually reaches 1. If it never reaches 1, it falls into an infinite cycle.

### Step-by-Step for 7 (Happy)

```
7  → 7² = 49
49 → 4² + 9² = 16 + 81 = 97
97 → 9² + 7² = 81 + 49 = 130
130 → 1² + 3² + 0² = 1 + 9 + 0 = 10
10 → 1² + 0² = 1 ✅
```

Since we reached 1, **7 is a happy number**.

### Step-by-Step for 2 (Not Happy)

```
2   → 2² = 4
4   → 4² = 16
16  → 1² + 6² = 1 + 36 = 37
37  → 3² + 7² = 9 + 49 = 58
58  → 5² + 8² = 25 + 64 = 89
89  → 8² + 9² = 64 + 81 = 145
145 → 1² + 4² + 5² = 1 + 16 + 25 = 42
42  → 4² + 2² = 16 + 4 = 20
20  → 2² + 0² = 4  ← cycle detected!
```

We're back at 4, which we already saw. This means we've entered a cycle that will never reach 1. So **2 is not a happy number**.

## Approach 1: HashSet (My Solution)

### Intuition

Use a HashSet to track all numbers we've seen. If we encounter a number we've already visited, we're in a cycle → return `false`. If we reach 1 → return `true`.

### Algorithm

1. Compute the sum of squares of digits of `n`.
2. If the result is in the HashSet → cycle detected, return `false`.
3. Otherwise, add it to the HashSet and repeat with the new number.
4. If `n` reaches 1, return `true`.

### Complexity Analysis

- **Time Complexity**: O(log n) per iteration (digits of n ≈ log n)
- **Space Complexity**: O(k) — stores all seen numbers until a cycle or 1 is found

### Code

```csharp
public static bool IsHappy(int n)
{
    HashSet<int> seen = new HashSet<int>();
    int tempNum = 0;
    while (n > 1)
    {
        while (n > 0)
        {
            int x = n % 10;
            tempNum += x * x;
            n /= 10;
        }
        if (seen.Contains(tempNum)) return false;
        seen.Add(tempNum);
        n = tempNum;
        tempNum = 0;
    }
    return (n == 1);
}
```

## Approach 2: Floyd's Cycle Detection (Optimal)

### Intuition

Treat the process as a linked list where each number points to the next. If there's a cycle, a slow and fast pointer will eventually meet. If the number is happy, the fast pointer will reach 1.

**Key insight**: If a number is not happy, the sequence will always cycle. Floyd's algorithm detects cycles in O(1) space.

### Algorithm

1. Initialize `slow = n`, `fast = GetNext(n)`.
2. Move `slow` one step and `fast` two steps at a time.
3. Stop when `fast == 1` (happy) or `slow == fast` (cycle).
4. Return `fast == 1`.

### Complexity Analysis

- **Time Complexity**: O(log n) per iteration
- **Space Complexity**: O(1) — no extra storage needed

### Code

```csharp
public bool IsHappy(int n) {
    int slow = n;
    int fast = GetNext(n);

    while (fast != 1 && slow != fast) {
        slow = GetNext(slow);
        fast = GetNext(GetNext(fast));
    }

    return fast == 1;
}

private int GetNext(int n) {
    int sum = 0;
    while (n > 0) {
        int digit = n % 10;
        sum += digit * digit;
        n /= 10;
    }
    return sum;
}
```
