## 📋 About This Repository

This repository documents my journey through LeetCode problems, organized by **algorithmic patterns** rather than arbitrary problem numbers. As a backend engineer, I focus on building strong fundamentals in data structures, algorithms, and problem-solving patterns that translate directly to real-world system design and optimization.

## 🎯 Approach

### Pattern-Based Learning
Instead of solving problems randomly, I group them by underlying patterns:
- **Arrays & Hashing**: Two pointers, sliding window, prefix sums
- **Linked Lists**: Fast/slow pointers, reversal, cycle detection
- **Trees & Graphs**: DFS, BFS, traversal variations
- **Dynamic Programming**: Memoization, tabulation, state machines
- **And more...** (see [PATTERNS.md](PATTERNS.md))

### Documentation Standard
Each problem includes:
- Problem statement and constraints
- Multiple solution approaches (brute force → optimized)
- Time and space complexity analysis
- Key insights and patterns identified
- Edge cases and testing strategy
- Code implementation with detailed comments

## 🔍 Problem Selection Strategy

1. **Foundation First**: Master easy problems to solidify fundamentals
2. **Pattern Recognition**: Focus on one pattern at a time
3. **Progressive Difficulty**: Graduate to medium/hard within each pattern
4. **Real-World Relevance**: Prioritize problems applicable to backend engineering
5. **Spaced Repetition**: Revisit difficult problems after intervals
